## 1. 프로젝트 개요

### 1.1 배경 및 목적

현재 학회, 기업, 관공서로부터 케이터링·단체도시락 문의가 지속적으로 유입되고 있으나, 문의 대응과 콘텐츠 소개가 개인 채널(전화, 메신저, SNS)에 분산되어 있어 다음과 같은 한계가 존재한다.

- 브랜드의 전문성과 레퍼런스를 체계적으로 보여줄 통합 창구 부재
- 문의 시 매번 견적 조건(인원, 예산, 형태 등)을 반복 확인해야 하는 비효율
- 대규모 행사 발주처(학회 사무국, 기업 총무, 관공서 담당자)가 신뢰할 수 있는 공식 웹사이트 부재
- 검색 유입(대구/경주 케이터링, 학회 도시락 등) 미확보

본 프로젝트의 목적은 **문의 전환을 극대화하는 B2B 케이터링 브랜드 웹사이트**를 구축하여, 흩어진 문의를 단일 창구로 집약하고 브랜드 전문성을 자산화하는 데 있다.

### 1.2 프로젝트 목표

**정성적 목표**
- 학회·기업·관공서 담당자가 "이 업체는 대규모 행사를 안심하고 맡길 수 있다"고 판단하게 만드는 신뢰 자산 구축
- 단순 음식 제공이 아닌, 경주·신라 헤리티지 스토리텔링과 프리미엄 실행 역량을 브랜드 차별점으로 각인

**정량적 목표** *(§2에서 KPI로 구체화)*
- 온라인 견적/상담 문의 폼을 통한 리드(lead) 확보
- 검색엔진(네이버·구글) 유입 확보를 위한 SEO 기반 구축
- 문의 응대 시간 단축(구조화된 견적 폼으로 사전 정보 수집)

### 1.3 롤모델 벤치마킹: 모나테이블 구조 분석

| 모나테이블 요소 | 특징 | 본 프로젝트 적용 |
|---|---|---|
| 영상 히어로 + 브랜드 슬로건 | "음식을 넘어 공간·분위기·감동을 디자인" | 경주 헤리티지 무드의 영상/이미지 히어로 |
| 서비스 4분류 | 출장 케이터링 / 푸드박스 / 박스 케이터링 / 음료 딜리버리 | MICE 케이터링 / 단체도시락 / 행사별 맞춤 등으로 재구성 |
| 파트너 로고 월 | 50여 개 브랜드 로고 노출로 신뢰 확보 | 학회·기업·관공서 레퍼런스 로고 월 |
| 포트폴리오 (브랜드별 상세 페이지) | 케이스 스터디 방식 | 대표 행사별 상세 케이스 페이지 |
| 현장스케치 (블로그) | 최신성·SEO 확보 | 행사 후기·현장 콘텐츠 발행 |
| 아카데미 | 교육 사업 확장 창구 | 쿠킹클래스·외국인 대상 클래스 소개 |
| 구조화된 견적 문의 폼 | 인원·예산·형태 등 사전 수집 | MICE 특화 항목 추가 |
| 멀티채널 퀵버튼 | 카카오톡·네이버톡톡·전화 | 동일 적용 |

### 1.4 핵심 차별화 포인트 (콘텐츠 전략)

1. **대규모 B2B 실행 역량** — 학회·기업·관공서 수백~수천 명 규모 행사 수행 레퍼런스
2. **경주·신라 헤리티지 스토리텔링** — 지역 문화를 담은 메뉴·연출로 차별화
3. **약 30년 경력 셰프의 조리 전문성** — 수비드·콩피 등 사전 생산 안정성이 높은 대량 조리 기법
4. **사전 생산 안정성** — 대량 행사에서도 품질을 유지하는 운영 시스템
5. **인바운드 외국인 단체 대응** — 다국어·할랄 등 특수 식이 요건 대응 역량

### 1.5 경쟁·벤치마크 확장 분석 *(신규)*

모나테이블 외 벤치마크를 넓혀 강점을 흡수하고 차별점을 명확히 한다.

| 구분 | 관찰 포인트 | 우리 사이트 시사점 |
|---|---|---|
| 브랜드 케이터링(모나테이블 등) | 감각적 비주얼·스타일링·포트폴리오 | 비주얼 완성도는 벤치마크, B2B 실적으로 차별화 |
| 대형 급식·단체도시락 업체 | 위생·HACCP·대량 납품 신뢰 강조 | 위생·정시납품·사전생산 안정성 전면화 |
| 호텔 연회/케이터링 | 격식·의전·프리미엄 연출 | 관공서·기업 의전 대응력 소구 |
| MICE 전문 에이전시 | 행사 통합 기획·다국어 대응 | 인바운드·다국어·특수식이 강점 연결 |

## 2. 성과 지표(KPI) 및 측정 체계 *(신규)*

목표를 "측정 가능한" 형태로 정의한다. 사이트 라이브 후 GA4·검색콘솔·리드 로그로 추적한다.

### 2.1 핵심 KPI

| KPI | 정의 | 초기 목표(런칭 후 3개월) | 측정 도구 |
|---|---|---|---|
| **월간 리드 수** | 문의 폼 제출 완료 건수 | 월 15~30건 | 폼 DB / GA4 이벤트 `generate_lead` |
| **문의 폼 전환율** | 폼 제출 ÷ CONTACT 페이지 방문 | 8% 이상 | GA4 퍼널 |
| **퀵버튼 전환** | 카카오·전화·톡톡 클릭 수 | 월 50클릭 이상 | GA4 이벤트 `contact_channel_click` |
| **유기 검색 유입** | 네이버·구글 오가닉 세션 | 월 500세션(3개월 후) | GA4 / 서치콘솔 |
| **핵심 키워드 노출·순위** | "경주 케이터링" 등 상위 노출 | 주요 5개 키워드 1페이지 진입 | 네이버 서치어드바이저·서치콘솔 |
| **성능(Core Web Vitals)** | LCP·INP·CLS 통과 비율 | 모바일 "양호" 75% 이상 | 서치콘솔 / PageSpeed |

### 2.2 전환 추적 이벤트 설계 (GA4)

개발 시 아래 이벤트를 심는다(이벤트명 고정, 파라미터 함께 전송).

| 이벤트명 | 트리거 | 주요 파라미터 |
|---|---|---|
| `generate_lead` | 문의 폼 제출 성공 | `inquiry_type`, `org_type`, `headcount_range` |
| `contact_channel_click` | 퀵버튼(카카오/전화/톡톡/인스타) 클릭 | `channel` |
| `cta_click` | 히어로·배너 견적 CTA 클릭 | `location`(hero/banner/service) |
| `view_case` | 레퍼런스 케이스 상세 조회 | `case_slug` |
| `service_detail_view` | 서비스 상세 페이지 조회 | `service_id` |
| `field_note_read` | 블로그 글 스크롤 70% 도달 | `post_slug` |

### 2.3 리드 품질 측정
- 폼 제출 시 `org_type`(발주처 유형)·`headcount_range`(인원 구간)·`budget_range`를 저장하여, 이후 **문의→견적→수주** 전환을 대표님이 수기 또는 CRM으로 태깅할 수 있게 한다(§18 CRM 확장 연계).

## 3. 타깃 및 브랜드 포지셔닝

### 3.1 타깃 고객 정의

| 세그먼트 | 담당자 | 니즈 | 핵심 소구점 |
|---|---|---|---|
| 학회·학술대회 | 사무국·운영대행사 | 대규모·다일정 도시락, 정시 납품, 위생 | 대량 행사 실적, 사전 생산 안정성 |
| 기업 | 총무·마케팅·행사담당 | 브랜드 격에 맞는 케이터링, 연출 | 프리미엄 실행력, 레퍼런스 |
| 관공서 | 행사·의전 담당 | 공신력, 명확한 견적·증빙 | 공식 사업자, 여성기업 인증 등 |
| 인바운드 단체 | 여행사·MICE 에이전시 | 지역성, 특수 식이(할랄 등) | 신라 스토리텔링, 다국어·특수식이 |

### 3.2 브랜드 구조

- **케이터링·단체도시락 (핵심 B2B)** — 학회·기업·관공서 대상 주력 서비스
- **프리미엄 헤리티지 다이닝·MICE 이벤트 (신라연 콘셉트)** — 예약제 프라이빗 다이닝·행사 (확장 소개)
- **쿠킹클래스·문화 체험** — 외국인·단체 대상 체험 프로그램

> 참고: 브랜드명·로고·상호 표기는 대표님 확정안에 따라 반영한다. 초기에는 케이터링 서비스를 전면에, 헤리티지 다이닝/클래스는 확장 메뉴로 배치할 것을 권장한다.

### 3.3 브랜드 메시지 및 톤앤매너

- **슬로건 방향(예시)**: "천년 고도의 품격을 한 상에 담다" / "학회부터 기업 행사까지, 대규모 행사의 완성"
- **톤**: 신뢰감 있는 프리미엄, 절제된 고급스러움 (과한 화려함 지양)
- **금지 톤**: 저가·박리다매 인상, 개인 맛집 감성

```
├── REFERENCES (레퍼런스 / 포트폴리오)
│     ├── 레퍼런스 로고 월 (학회·기업·관공서)

└── [글로벌] 언어 전환 (KO / EN)
```

**전역 요소**
- 상단 GNB(글로벌 내비게이션) + 로고
- 우측 하단 퀵버튼: 카카오톡 채널 / 네이버 톡톡 / 온라인 견적 / 전화 / 인스타그램
- 하단 푸터: 사업자 정보, SNS, 이용약관·개인정보처리방침
- **쿠키/분석 동의 배너**(GA4·개인정보 처리 고지) — §9 참조 *(신규)*
- **404 / 500 에러 페이지**, 접근성 스킵 내비게이션 *(신규)*

## 5. 페이지별 상세 기획

### 5.1 HOME (메인)

| 섹션 | 구성 요소 |
|---|---|
| 히어로 | 풀스크린 영상/이미지 + 브랜드 슬로건 + 견적 CTA 버튼 |
| 서비스 요약 | 4개 서비스 카드(아이콘·1줄 카피·상세 링크) |
| 레퍼런스 신뢰 | 학회·기업·관공서 로고 월 (무한 슬라이드) |
| 대표 케이스 | 주요 행사 포트폴리오 3~6건 카드 |
| 차별점 | 대규모 실행력·헤리티지·조리 전문성 3~4포인트 |
| 현장스케치 | 최신 글 3건 카드 |
| CTA 배너 | "행사를 준비 중이신가요?" 견적 유도 |
| 문의 폼 | 하단 간이 견적 문의 폼(또는 CONTACT 링크) |

### 5.2 ABOUT (브랜드 소개)
- 브랜드 스토리(경주·신라 문화 기반 스토리텔링)
- 약 30년 경력 셰프 및 운영 파트너십 소개
- 보유 인증·자격(여성기업 인증 등)·연혁 타임라인

### 5.3 SERVICES (서비스)
각 서비스는 **개별 상세 페이지**로 구성하며 공통 템플릿을 따른다.
- 서비스 정의 및 적합 대상
- 제공 범위(메뉴 유형, 세팅·스타일링, 운영 인력, 사후 정리 등)
- 규모별 안내(소규모 / 대규모 수백~수천 명)
- 대표 메뉴·구성 예시(사진)
- 특수 요건 대응(할랄·채식 등)
- 해당 서비스 견적 CTA

**서비스 항목**
1. **MICE·행사 케이터링** — 학회·기업·관공서 출장 케이터링(뷔페·핑거푸드·플레이팅)
2. **단체도시락 / 프리미엄 도시락** — 다일정 학회 대응, 사전 생산 안정 메뉴
3. **헤리티지 다이닝·프라이빗 이벤트(신라연)** — 예약제 프리미엄 다이닝·기업 MICE 만찬(확장 소개)
4. **쿠킹클래스·문화 체험** — 외국인·단체 대상 체험 프로그램

### 5.4 REFERENCES (레퍼런스 / 포트폴리오)
- **로고 월**: 발주처 로고 그리드 (기밀 이슈 시 업종·규모만 표기하는 방식도 지원)
- **케이스 상세**: 행사별 개별 페이지 (개요·규모·과제·솔루션·결과·현장 사진 갤러리)
- 필터: 발주처 유형(학회/기업/관공서), 규모, 형태
- **주의**: 로고·사진 사용은 발주처 동의 필요. 미동의 시 "OO학회(1,200명)"식 텍스트 표기 대체안 제공.

### 5.5 FIELD NOTES (현장스케치 / 블로그)
- 목록(썸네일·제목·요약·날짜) + 상세(리치 콘텐츠)
- 카테고리·태그, SEO 메타(제목·설명·OG 이미지) 개별 설정
- 검색 유입 및 최신성 확보 목적

### 5.6 CONTACT (견적·상담 문의) — **최우선 전환 페이지**

**폼 항목**
- 업체명/기관명 (필수)
- 담당자 성함 (필수)
- 연락처 (필수)
- 이메일
- 발주처 유형: 학회 / 기업 / 관공서 / 여행사·MICE / 기타 (선택)
- 행사 유형: 케이터링 / 단체도시락 / 프라이빗 다이닝 / 쿠킹클래스 / 기타
- 행사 일자 / 시간
- 행사 인원
- 행사 장소
- 1인 예산 또는 총예산
- 특수 요건: 할랄 / 채식 / 알레르기 / 기타 (다중 선택)
- 문의 내용 (자유 서술)
- **개인정보 수집·이용 동의 (필수 체크) + 처리방침 링크** *(문구 §9 명세)*

**동작·검증** *(보강)*
- 클라이언트+서버 양쪽 유효성 검증(필수값·이메일/전화 형식)
- 제출 시 대표 이메일로 알림 발송 + 관리자 DB 저장(리드 백업)
- **자동 회신 메일**(제출자에게 "접수 완료 + 예상 회신 시간") — 선택
- 스팸 방지: reCAPTCHA v3(또는 hCaptcha) + **허니팟 필드** + **서버측 rate limiting**
- 제출 실패 시 재시도 안내(네트워크·서버 오류 구분)
- 제출 완료 화면 + `generate_lead` 이벤트 발화

## 8. 기술 스택·아키텍처·데이터 모델

### 8.1 권장 스택 (Primary)

| 영역 | 기술 | 사유 |
|---|---|---|
| 프레임워크 | **Next.js (App Router) + TypeScript** | SSR/SSG로 SEO 강점, 유지보수 용이 |
| 스타일 | **Tailwind CSS** | 빠른 반응형 구현, 일관된 디자인 토큰 |
| 콘텐츠 관리 | **Sanity(헤드리스 CMS)** 또는 MDX | 비개발자가 레퍼런스·블로그 직접 관리 가능 |
| 문의 폼 처리 | API Route + **Resend**(이메일) + DB | 즉시 알림 + 리드 보관 |
| 리드 저장 | **Vercel Postgres / Supabase / Google Sheets(경량)** | 리드 백업·조회 |
| 배포 | **Vercel** | Next.js 최적, 자동 배포·SSL |
| 분석 | GA4, 네이버 서치어드바이저 | 유입·전환 측정 |
| 에러 모니터링 | **Sentry**(선택) | 폼 오류·런타임 예외 추적 |

### 8.2 대안 스택 (경량)
- 콘텐츠 업데이트 빈도가 낮다면 **Next.js + MDX(파일 기반)** 로 시작하고 이후 CMS 도입.
- 리드 저장도 초기엔 이메일 알림 + Google Sheets 연동으로 시작 가능(비용 최소화).

### 8.3 콘텐츠 데이터 모델 *(신규 — CMS/MDX 공통 스키마)*

## 9. 개인정보·법적 준수 *(신규 · 필수)*

문의 폼으로 개인정보(성명·연락처·이메일)를 수집하므로 **개인정보보호법·정보통신망법** 준수 대상이다.

### 9.1 필수 준수 항목
- **개인정보 수집·이용 동의**: 폼 제출 전 필수 체크(사전 체크 상태 금지). 수집항목·목적·보유기간·거부권리 명시.
- **개인정보처리방침 페이지**(`/privacy`): 수집 항목/목적/보유기간/파기/처리위탁(Resend·Vercel·GA)/이용자 권리/개인정보 보호책임자 연락처.
- **이용약관 페이지**(`/terms`).
- **보유기간·파기**: 문의 목적 달성 후 보유기간 명시(예: 1년) 후 파기.
- **처리위탁 고지**: 이메일(Resend), 호스팅(Vercel), 분석(Google)의 국외 이전 포함 고지.
- **쿠키/분석 동의**: GA4 등 분석 쿠키 사용 시 고지·동의 배너.
- **사업자 정보 표기**(전자상거래법): 푸터에 상호·대표자·사업자등록번호·주소·연락처.

### 9.2 동의 문구(예시 — 법무 검토 후 확정)
> "귀하가 입력하신 정보(기관명·성명·연락처·이메일·행사정보)는 견적 상담 및 회신 목적으로 수집·이용되며, 목적 달성 후 [1년]간 보관 후 파기됩니다. 동의를 거부할 권리가 있으나, 거부 시 상담 진행이 제한될 수 있습니다. [개인정보처리방침 전문 보기]"

## 10. 보안·인프라 요구사항 *(신규)*

### 10.1 문의 폼·API 보안
- 서버측 유효성 검증 + reCAPTCHA v3 스코어 검증 + 허니팟 필드

- 리드는 이메일 실패에 대비해 **DB에도 이중 저장**

### 10.3 인프라·운영
- HTTPS 강제(Vercel 자동 SSL), 보안 헤더(CSP·HSTS 등) 설정
- 도메인 등록·DNS 관리 주체 명확화(대표님 소유 권장)
- 콘텐츠·리드 데이터 백업 정책(DB 자동 백업 또는 주기적 export)
