# 관리자 페이지 설정 가이드 (Cloudflare)

> 코드는 GitHub에 올라가 있고 Cloudflare Pages에 자동 배포됩니다.
> **관리자 페이지가 실제로 작동하려면 아래 3가지 설정을 Cloudflare 대시보드에서 한 번만 해주시면 됩니다.**

관리자 페이지 주소: **https://catering-wedsite-2.pages.dev/admin.html**

---

## 1단계 · D1 데이터베이스 만들기 (문의가 저장될 곳)

1. Cloudflare 대시보드 접속 → 왼쪽 메뉴 **Workers & Pages → D1 SQL Database**
2. **Create database** 클릭 → 이름 입력(예: `silla-inquiries`) → 생성
3. 만든 DB 클릭 → **Console(콘솔)** 탭 → 아래 내용을 붙여넣고 **실행(Execute)**

```sql
CREATE TABLE IF NOT EXISTS inquiries (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at TEXT,
  org_name TEXT,
  contact_name TEXT,
  phone TEXT,
  email TEXT,
  org_type TEXT,
  event_type TEXT,
  event_date TEXT,
  event_time TEXT,
  headcount TEXT,
  location TEXT,
  budget TEXT,
  message TEXT,
  status TEXT DEFAULT '신규'
);
```
(이 SQL은 저장소의 `schema.sql` 파일에도 있습니다.)

---

## 2단계 · Pages 프로젝트에 DB 연결(바인딩)

1. **Workers & Pages → (catering-wedsite-2 프로젝트 선택)**
2. **Settings(설정) → Functions → D1 database bindings** (또는 Bindings)
3. **Add binding** 클릭
   - **Variable name(변수명)**: `DB` ← 반드시 대문자 DB
   - **D1 database**: 1단계에서 만든 `silla-inquiries` 선택
4. 저장

---

## 3단계 · 관리자 비밀번호 설정

1. 같은 프로젝트 → **Settings → Environment variables(환경 변수)**
2. **Add variable** 클릭
   - **Variable name**: `ADMIN_PASSWORD`
   - **Value**: 원하시는 비밀번호 (예: 강력한 비밀번호로)
   - **Production** 환경에 추가
3. 저장

---

## 4단계 · 재배포

- 설정을 저장한 뒤, 프로젝트 **Deployments** 탭에서 최신 배포를 **Retry deployment**(재배포) 하거나,
- 코드가 다시 push되면 자동으로 재배포됩니다. (설정 반영을 위해 한 번 재배포 권장)

---

## ✅ 사용 방법

1. **https://catering-wedsite-2.pages.dev/admin.html** 접속
2. 3단계에서 정한 **비밀번호로 로그인**
3. 홈페이지에서 들어온 견적 문의가 **표로 최신순** 정렬되어 보입니다
4. 각 문의의 **상태(신규 → 응대중 → 견적발송 → 완료)** 를 드롭다운으로 관리
5. 연락처·이메일은 클릭 시 바로 전화/메일 연결

---

## 자주 묻는 질문

**Q. 로그인 화면에 "ADMIN_PASSWORD가 설정되지 않았습니다"가 떠요.**
→ 3단계(환경 변수)가 안 됐거나 재배포 전입니다. 3~4단계를 확인하세요.

**Q. 문의를 넣었는데 관리자에 안 보여요.**
→ 1·2단계(DB 생성·바인딩)를 확인하세요. 변수명이 정확히 `DB`여야 합니다.

**Q. 이메일 알림도 받고 싶어요.**
→ 가능합니다. 추후 Resend 등 메일 서비스 키를 연동하면 문의 접수 시 대표 이메일로 알림을 보낼 수 있습니다. 필요하시면 요청해 주세요.

**Q. 보안은 괜찮나요?**
→ 관리자 데이터는 로그인(비밀번호)한 사람만 조회할 수 있도록 서버에서 보호됩니다. 비밀번호는 강력하게 설정하고 공유하지 마세요.
